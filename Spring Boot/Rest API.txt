Rest Api

  路径， 又称为重点，（endpoint），表示API的具体网址

  RESTful 架构中，每一个网址代表一种资源，（resource），所有网址中不能有动词，只能有名词，
      而且所有的名字基本都与数据库里的名词所对应。


HTTP 动词：
  GET （SELECT）：从服务器取出一项或者多项资源 
  POST （CREATE）：建立一个资源在服务器中
  PUT （UPDATE）： 在服务器中更新资源 （客户端提供给变后完整的资源）
  PATCH （UPDATE）：在服务器中更新资源 （客户端提供更改的属性）
  DELETE （DELETE） ： 从服务器中删除资源
